## 2. Library Utilities (`src/lib/`)

### `notebooksFs.js` — Build-time filesystem reader (Node only)
- `readNotebooksFromDisk(docsDir)` — walks `docs/`, returns sorted array of notebooks
- Each notebook: `{ dir, label, position, hasMetadata, notes: [{ file, id, title, updated }] }`
- Reads `_category_.json` for label/position; reads `_metadata.json` for note order/titles
- Skips empty notebooks, hidden files (`_` prefix), non-markdown files
- Used only by `sidebars.js` and `notebook-snapshot` plugin (never in browser)

### `notes.js` — Shared pure helpers (browser + Node)
- Constants: `OWNER`, `REPO`, `BRANCH`, `DOCS_PATH`, `API` (GitHub API endpoint)
- `slugify(text)` — text → URL-safe slug
- `b64Encode/b64Decode` — Base64 via `btoa`/`atob`
- `parseMetadata(obj)` — handles both new (`{ titles, order, updated }`) and legacy flat formats
- `serializeMetadata(titles, order, updated)` — serializes to JSON string
- `noteUpdatedAt(name, updated)` — extracts timestamp from metadata or filename suffix (`slug-<epochMs>.md`)
- `formatTimestamp(ms)` — epoch ms → locale string (e.g. "Jun 7, 2026, 3:42 PM")
- `orderNotes(notes, order)` — sorts by saved order; appends unknown files at end
- `moveToIndex(list, item, index)` — reorders an item in a list

### `notes.d.ts` — TypeScript ambient declarations for `notes.js`
- Declares all exports from `notes.js` for browser-side TypeScript
- Imports `NoteMetadata` type from `@site/src/types`

## 3. Custom Hooks (`src/hooks/`)

### `useGitHubNotebook.ts` — Central data + control layer
- **State (18 properties):** notebooks, selectedNotebook, notes, metadata, view (`list`/`edit`/`new-notebook`), editingNote, loading flags, saving, syncing, syncProgress, refreshing, refreshProgress, conflictBanner, saveModal, notebookModal
- **Operations (20):** fetch/select/create notebooks; open/create/save/delete/reorder notes; conflict handling; modal controls
- Side effects: polling GitHub API to confirm writes propagated, auto-dismiss modals, sync intervals/timeouts
- Delegates network calls to `GitHubNotebookClient`; uses `useNotifications` for error reporting

### `useNoteListInteractions.ts` — Note-list UI interaction reducer
- `useReducer`-based; resets all state when notebook changes
- **State:** `expanded` (Set), `confirmingDelete`, `dragName`, `dropIndex`, `hoverIndex`, `draftAfter`, `activeInsert`
- **Returns:** state + 11 action creators: `toggleExpand`, `confirmDelete`, `cancelDelete`, `dragStart`, `dragEnd`, `setDropIndex`, `setHoverIndex`, `startDraft`, `closeDraft`, `setActiveInsert`
- No external APIs; pure client-side state machine

### `useNotifications.tsx` — App-wide toast notification system
- Context provider with `toasts` array and `nextId` ref
- Auto-dismiss timers: success 3s, info 4s, error 7s
- API: `notify(message, kind)` and `dismiss(id)`
- Renders ARIA live region (`aria-live`, `aria-atomic`) for accessibility

### `useSaveLifecycle.ts` — Single-note save state machine
- States: `idle` | `saving` | `committed` | `error`
- Auto-resets to `idle` after 1.5s on success
- API: `begin()`, `succeed()`, `fail(error)`, `reset()`, plus derived `saving` and `justSaved` booleans
- Prevents impossible states (e.g. simultaneous `saving` + `error`)

## Key Architectural Patterns

| Pattern | Where |
|---|---|
| GitHub as backend | All note CRUD via GitHub Contents API (`GitHubNotebookClient`) |
| Build-time snapshot + live fetch | `notebook-snapshot` plugin seeds homepage; live fetch overlays on mount |
| SHA-based conflict detection | `ExpandableNote`, `NoteEditor` track file SHA; GitHub rejects stale writes |
| State machine hooks | `useSaveLifecycle` (idle/saving/committed/error), `useNoteListInteractions` (reducer) |
| Lazy content loading | `ExpandableNote` fetches body only on first expand |
| Metadata file per notebook | `_metadata.json` stores `{ titles, order, updated }` alongside the markdown files |
| Timestamp in filename | Notes without metadata entry fall back to `<slug>-<epochMs>.md` suffix for `updatedAt` |
| SSR safety | `MarkdownPreview` wrapped in `BrowserOnly`; `write.tsx` hydrates PAT from localStorage after mount |
